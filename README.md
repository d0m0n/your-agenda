# あなた次第(Your agenda)

青年会議所(JC)の委員会運営を楽にするWebサービスです。委員会組織ごとに契約し、
会議・次第・メンバー・資料を一元管理できます。アップロードしたZipファイル内の
HTML議案(gian.htm)や、PDF・画像単体の議案ファイルをURLひとつで共有できます。

サービス名の由来:「議題(agenda)はあなた次第」

## 利用ユーザーの種類

| 種別 | 説明 | できること |
| --- | --- | --- |
| 一般ユーザー | 委員会組織ごとに契約するアカウント | 組織情報・メンバー・役職・会議・次第・資料の全CRUD、オブザーブユーザーの管理、基本設定の変更 |
| オブザーブユーザー | 一般ユーザーが作成する閲覧専用アカウント | ダッシュボード・会議一覧・会議詳細・メンバー一覧・資料の閲覧のみ(作成・編集・削除・CSV入出力は不可、個人プロフィール編集も不可) |
| 管理者アカウント(super_admin) | 運営側がプラットフォーム全体を管理するための、組織に属さないアカウント | 全組織の契約状況・データ使用量の閲覧、ユーザーごとの容量変更、アップロード済みデータの削除、アカウントの削除 |

一般ユーザーはランディングページ(`/lp`)経由で自分自身が組織登録できます
(下記「ランディングページ・組織登録」参照)。オブザーブユーザーは一般
ユーザーが基本設定画面から作成し、管理者アカウントはサーバー管理者が
`php artisan admin:create-super-admin` コマンドで作成します。

## 画面一覧と使い方

### ランディングページ・組織登録

- 未ログインで利用できるランディングページ(`/lp`)から、無料で組織登録が
  できます。「無料ではじめる」から`/register`へ進み、組織名・お名前・
  メールアドレス・パスワードを入力すると、組織(organization)と一般
  ユーザーがセットで作成され、そのまま自動ログインしてダッシュボードへ
  移動します。
- 未ログインで`/`にアクセスした場合は、従来どおり`/login`へ自動
  リダイレクトされます(LP自体は`/`を差し替えず`/lp`で個別に公開)。

### ログイン・パスワード再設定

- ログイン画面(`/login`)からメールアドレスとパスワードでログインします。
- 「パスワードを忘れた場合」からパスワード再設定メールを送信できます(`/forgot-password`)。

### ダッシュボード(トップページ)

ログイン後の最初の画面。組織のヘッダー画像の下に、最大4つのペインを表示します。
一般ユーザー・オブザーブユーザーどちらも閲覧できます。

| ペイン | 内容 |
| --- | --- |
| 今後の会議予定 | 本日以降に開催予定の会議を開催日時の昇順で表示(開催日時・開催場所・会議画面へのリンク)。該当する会議がなければ「予定なし」と表示 |
| カレンダー | 基本設定で登録したGoogleカレンダーをiframeで埋め込み表示 |
| 今月の誕生日 | メンバーの生年月日から当月分を抽出して表示 |
| 資料置き場 | 資料の一覧とダウンロードリンク(直近5件) |

各ペインの表示・非表示は基本設定画面から切り替えられます。ペインが奇数個
表示される場合、最後の1つは2カラム分に広がって表示されます。

**制限**: 管理者アカウント(super_admin)はダッシュボードを利用しません。ログイン後は
自動的に管理者パネルへリダイレクトされます。

### 会議管理

会議の一覧・作成・編集・詳細画面です。一覧・詳細はオブザーブユーザーも閲覧できますが、
作成・編集・削除は一般ユーザーのみです。

#### 会議一覧(`/meetings`)

会議名・開催日時・開催場所の一覧です。一般ユーザーには新規登録・編集・削除リンクが、
オブザーブユーザーには「詳細」リンクのみが表示されます。各列の見出しをクリックすると
その項目でソートできます(もう一度クリックすると昇順・降順が切り替わります)。
デフォルトは開催日時が新しいものから順(降順)です。

会議の管理は目的別に2つの画面に分かれています(いずれも一般ユーザーのみ)。

#### 会議情報編集(`/meetings/{id}/edit`)

以下の項目を設定できます。

- 会議名(必須)
- 開始日時・終了日時(どちらも任意。終了日時は開始日時より後である必要があります)
- 開催場所・開催場所の住所・地図URL(GoogleマップのURL等)
- 懇親会・2次会情報、推奨ホテル情報(自由記述、案内文に差し込めます)
- Wi-Fi SSID・パスワード(閲覧権限のある組織メンバーのみ表示されます)
- メモ
- 会議ヘッダー画像(jpg/png/webp、自動でリサイズ・再エンコードして保存)

#### 次第編集(`/meetings/{id}/agenda`)

- **議案ファイルのアップロード**: Zip、またはPDF・画像(jpg/jpeg/png/gif/webp)
  単体をアップロードできます。Zipの場合は `gian.htm` をZip直下または1階層下の
  フォルダに配置してください。PDF・画像は単体アップロードのためファイル名を
  問わずそのまま公開されます。アップロード中は進捗バーが表示されます。
- **議案ファイルの差し替え**: アップロード済みの議案ファイル一覧から「差し替え」を
  選ぶと、公開URL・タイトル・次第からのリンクはそのままに、ファイルの中身だけを
  新しいものに置き換えられます。差し替え用のファイルが不正な場合(壊れたZip等)は
  エラーになり、既存のファイルはそのまま残ります(差し替えに失敗しても壊れません)。
- **次第(アジェンダ)の追加・編集・削除・並び替え**: タイトル・担当者(登録済み
  メンバーまたは自由入力の氏名)・議案データのリンクを設定できます。項目は
  1階層だけ子項目を持てます(子項目にはさらに子項目を追加できません)。
- **議案データのリンク**: 次第の各項目には、この会議専用にアップロードした
  議案ファイル(sites)だけでなく、資料管理(資料置き場)に登録済みの組織内
  共有の資料(materials)もリンクできます。1つのセレクトに「議案ファイル
  (この会議)」「資料置き場(組織共有)」がグループ分けして表示され、どちらか
  一方を選べます。会議を跨いで使う規約・様式などは資料置き場に登録しておけば、
  複数の会議の次第から同じ資料にリンクできます。
- **過去の次第からコピー**: 他の会議の次第(トップレベル項目とその子項目)を
  選んで、この会議の次第の末尾にまとめて追加できます。会議専用の議案ファイル
  (sites)のリンクは会議ごとに紐付くためコピーされませんが、組織内共有の資料
  (materials)のリンクは一緒にコピーされます。コピー元として選べるのは「次第が
  1件以上登録されている、この会議以外の会議」のみです。該当する会議がない
  場合、この欄自体が表示されません。

#### 会議詳細画面(次第、`/meetings/{id}`)

会議名・開催日時・開催場所・次第・Wi-Fi情報・メモを表示します。次第の議案ファイル
リンクをクリックすると議案を確認できます。一般ユーザーには「印刷」ボタンと、
会議情報編集・次第編集それぞれへのリンクが表示されます。

- **印刷**: 印刷時はナビゲーション・ヘッダー画像・操作ボタンを自動的に隠し、
  次第シートだけを白黒・A4基準のレイアウトで印刷します。ブラウザの印刷
  ダイアログから「PDFとして保存」を選べばPDF化もできます。

### 次第の公開共有リンク

会議画面から、ログイン不要でアクセスできる外部共有リンクを発行できます
(一般ユーザーのみ)。

- 「発行する」を押すとURL(`/s/meetings/{token}`)が発行され、コピーして
  メール・LINE等で共有できます。漏洩時などは「再発行」でURLを差し替え、
  不要になったら「無効化」で古いURLを使えなくできます。
- 公開ページにはWi-Fi情報・メモは表示されません(組織メンバー限定の
  情報のため)。次第からリンクされている議案ファイル・資料も、同じ
  リンクだけで(ログインなしに)確認できます。

### 案内文作成

会議画面(または会議一覧)から「案内文作成」に進むと、PDF・メール・LINE
本文の3種類の案内文を、会議情報から自動生成した状態で編集できます。

- 自動生成される文面には、会議名・日時・場所・議題一覧などが差し込まれます。
  手直しして保存すると次回開いたときも編集内容が残り、「テンプレートに
  戻す」で自動生成の状態に戻せます。
- 基本設定画面(下記)で、組織ごとにテンプレートの既定文を編集できます。
- PDF案内文は画面の「印刷 / PDF保存」からブラウザの印刷機能でPDF化します
  (専用ライブラリでのPDF生成は行っていません)。地図URLを設定した会議は、
  QRコードが自動でPDFに掲載されます。
- PDF案内文のテンプレートでは、行の先頭に `>>` を付けるとその行を右揃えに、
  行に `[改ページ]` とだけ書くとそこで強制的に改ページできます。

### 役職管理

`/positions` — 会長・副会長など、委員会内の役職マスタを管理します。表示順を
決める通し番号(組織内で一意)と役職名を登録し、メンバー登録時に紐付けます。

### メンバー管理

`/members` — 会議の出欠管理や誕生日表示、次第の担当者選択に使うメンバー名簿です。
ログインアカウントとは別の概念で、メンバー自身がログインすることはありません。

- 必須項目は氏名のみ。よみがな・ローマ字表記・生年月日・性別・役職・所属企業・
  電話番号・メールアドレス・LINE ID・各種SNSアカウント・趣味・座右の銘・
  通し番号(組織内一意)・顔写真は任意です。
- 顔写真はjpg/png/webpのみアップロード可能で、自動でリサイズ・再エンコードされます。
- 一覧は氏名・よみがな・所属企業・生年月日・通し番号・役職でソートできます。
- **CSV一括登録**: テンプレートをダウンロードして入力し、アップロードすると
  一括登録できます(顔写真はCSVの対象外)。
- **CSVエクスポート**: 登録済みメンバー全員をCSVでダウンロードできます。
- 一覧の閲覧はオブザーブユーザーも可能ですが、新規登録・編集・削除・
  CSV一括登録/エクスポートは一般ユーザーのみです。
- 一覧は表形式・カード形式を切り替えられます(選択状態はブラウザに保存)。
  行(カード)をクリックすると、そのメンバーのプロフィール画面
  (`/members/{id}`、名刺風デザイン)に遷移します。プロフィール画面には
  前後のメンバー(氏名順)へ移動するリンクがあります。

### 資料管理(資料置き場)

`/materials` — 総会資料や規約など、会議に紐付かない資料を保管する場所です。

- アップロード可能: pdf, doc(x), xls(x), ppt(x), zip, jpg, png, gif, webp,
  txt, csv。最大20MB。
- アップロードには進捗バーが表示されます。
- **差し替え**: 一覧の各資料から「差し替え」を選ぶと、タイトル・ダウンロード
  URL・次第からのリンクはそのままに、ファイルの中身だけを新しいものに
  置き換えられます。差し替え用のファイルが不正な場合はエラーになり、
  既存のファイルはそのまま残ります(差し替えに失敗しても壊れません)。
- 閲覧・ダウンロードはオブザーブユーザーも可能ですが、アップロード・差し替え・
  削除は一般ユーザーのみです。

### 基本設定

`/settings` — 一般ユーザーのみアクセスできます。

- **データ使用量**: 資料・議案ファイル・各種画像の合計使用量を、割り当て容量
  (デフォルト2GB)に対する割合とともにプログレスバーで表示します。
- **組織情報**: 組織名、GoogleカレンダーID、ダッシュボードのヘッダー画像・
  アイコン画像(jpg/png/webp、自動リサイズ)を編集します。
- **ダッシュボード表示設定**: 4つのペイン(会議一覧・カレンダー・誕生日・
  資料置き場)の表示・非表示を切り替えます。
- **案内文のデフォルト**: PDF・メール・LINE案内文のテンプレートを、組織
  全体の既定として編集できます(プレースホルダー入り)。未編集の項目は
  組み込みの既定テンプレートが使われます。会議ごとの案内文作成画面は、
  ここで設定した内容を初期値として開きます。
- **オブザーブユーザー管理**: 専用画面(下記)への入口です。
- **次第の一括ダウンロード**: 組織の全会議・次第・議案ファイルをZipにまとめて
  ダウンロードします(会議ごとにフォルダ分け、次第はHTMLで書き出し)。解約時の
  データ持ち出し手段としても利用できます。

### オブザーブユーザー管理

`/observers` — 一般ユーザーが、閲覧専用アカウントを作成・編集・削除します。

- 作成時はログインID(メールアドレス)とパスワードが必須です。
- 編集時にパスワード欄を空欄のままにすると、パスワードは変更されません。
- ログインIDはシステム全体で一意である必要があります(他組織・他ユーザーと重複不可)。

### 管理者パネル(super_admin)

`/admin` — 運営側の管理者アカウント専用の画面です。通常のログイン画面から
ログインしますが、ログイン後は専用レイアウトで管理者パネルが表示されます。
一般ユーザー・オブザーブユーザーはアクセスできません。

- **組織一覧**: 全組織の契約ステータス・契約日・ユーザー数・データ使用量を一覧表示します。
- **組織詳細**: 契約状況の確認、組織に属するユーザー(一般/オブザーブ)の一覧、
  一般ユーザーごとの割り当て容量(GB単位)の変更、アカウントの削除ができます。
- **アップロード済みデータの削除**: その組織の資料・議案ファイル・各種画像を
  すべて削除して使用量をゼロに戻します。会議・メンバー等のレコード自体は
  削除されません(業務データを残したまま、ストレージだけ解放する操作です)。
- 契約ステータス自体の変更・組織の完全削除・課金操作は未実装です。

## 制限事項一覧

| 項目 | 制限内容 |
| --- | --- |
| 議案Zip | 展開後合計200MB以下、ファイル数1000件以下。拡張子ホワイトリスト方式(html, htm, css, js, png, jpg, jpeg, gif, svg, webp, ico, woff, woff2, ttf, json, txt, pdf, mp4 のみ展開、.php等は展開しない) |
| 議案ファイル単体 | PDF・jpg・jpeg・png・gif・webpのみ、最大200MB |
| 資料(materials) | pdf, doc(x), xls(x), ppt(x), zip, jpg, png, gif, webp, txt, csv、最大20MB |
| 各種画像(ヘッダー・アイコン・顔写真) | jpg/png/webpのみ、最大5MB。アップロード後は自動でリサイズ・再エンコードされる |
| 次第の階層 | 親子1階層のみ(子項目にさらに子項目は作れない) |
| 次第のコピー | 議案ファイルのリンクはコピーされない。コピー元は次第が1件以上ある他会議のみ選択可 |
| データ容量 | 一般ユーザー1人あたりデフォルト2GB(管理者パネルからユーザーごとに変更可)。資料・議案ファイル・各種画像の合計で判定し、超過時は新規アップロードを拒否(既存データは保持される) |
| オブザーブユーザー | 閲覧のみ。作成・編集・削除・CSV入出力は一般ユーザーのみ |
| ログインID(メールアドレス) | 全ユーザーを通してシステム全体で一意 |
| マルチテナント | 全データは組織(organization)単位でスコープされ、他組織のデータには一切アクセスできない |
| 地図URL | 有効なURL形式のみ、最大2048文字 |
| 懇親会・2次会情報/推奨ホテル情報 | 自由記述、最大2000文字 |

## 開発環境のセットアップ

- Docker + Laravel Sail(PHP 8.3固定)、DBはMySQLを使用します。
- 初回セットアップの概要:

  ```bash
  composer install
  cp .env.example .env
  php artisan key:generate
  ./vendor/bin/sail up -d
  ./vendor/bin/sail artisan migrate
  ./vendor/bin/sail artisan storage:link
  npm install && npm run build
  ```

- 一般ユーザーは `/lp` → `/register` からセルフサービスで組織登録できます
  (組織+一般ユーザーがセットで作成されます)。動作確認用に手早くアカウントを
  作りたい場合は、従来どおり `tinker` やシーダーでも作成できます。
- オブザーブユーザーは、一般ユーザーでログインして基本設定画面から作成します
  (UIからの新規登録経路はありません)。
- 管理者アカウント(super_admin)は以下のコマンドで作成します。

  ```bash
  ./vendor/bin/sail artisan admin:create-super-admin "管理者名" admin@example.com "パスワード"
  ```

- フロントエンド(Tailwind CSS)を変更した場合は `npm run build`
  (開発中の自動反映は `npm run dev`)を実行してください。

## デプロイの注意点

本番環境はさくらのレンタルサーバー(Apache、SSH可、root権限なし、Node.js無し)
を想定しています。詳細な制約・手順は `CLAUDE.md` の「本番環境の制約」を
参照してください。特に見落としやすい点だけ、ここに要点をまとめます。

通常のデプロイフローは `git pull` → `composer install --no-dev` →
`php artisan migrate` → (`public/build/` を手動アップロード)です。

1. **フロントエンド資材は手動アップロード**: `public/build/` は
   `.gitignore` 対象のため `git pull` では配置されません。デプロイ前に
   ローカルで `npm run build` を実行し、生成された `public/build/` を
   FTPS/SFTP等でサーバーへ手動アップロードしてください。忘れると
   Viteのmanifestが見つからずアプリ全体が500エラーになります。
2. **`storage:link` は初回デプロイ時に1回だけ**: `php artisan storage:link`
   を実行しないと、議案ファイル・各種ヘッダー画像・顔写真などが軒並み
   404になります。`git pull` では作られないシンボリックリンクなので、
   通常のデプロイフローとは別に、サーバー移行時など最初の1回だけ実行が
   必要です。
3. **nginx設定は使えません**: Webサーバーは常にApacheのため、アクセス
   制御・リダイレクト等はすべて `.htaccess` で行います(nginx用の設定を
   書いても反映されません)。
4. **Laravelスケジューラ用のcron**: サーバーのコントロールパネルから
   `php artisan schedule:run` を毎分(または最短間隔)で登録してください。
